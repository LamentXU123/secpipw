介绍

secured_pip 是一个轻量的 pip wrapper。它保持普通 pip 的行为，只在 pip install 周围加入供应链安全检查。

概览

secured_pip 面向日常 Python 安装路径设计。除 install 之外的命令都会转发给 python -m pip。对于 spip install 这个 CLI 入口，解析、下载、构建和安装仍然走 pip；secured_pip 只在关键节点插入轻量检查。

安装

先安装 secured_pip，再把 pip alias 到 spip，这样原来的工作流不用改变。

cmd（Windows）

pip install secured_pip
doskey pip=spip $*

Bash（Linux）

pip install secured_pip
echo "alias pip='spip'" >> ~/.bashrc
source ~/.bashrc

Zsh（macOS）

pip install secured_pip
echo "alias pip='spip'" >> ~/.zshrc
source ~/.zshrc

按照上述命令安装完成后，你可以通过运行 pip install --spip-status 去检查 spip 是否被成功安装。如果安装成功，你将会看到 spip {VERSION} guard enabled. 这条信息

报警机制

secured_pip 发现的每个风险都会被归入三个等级之一。在默认 low 敏感度下， HIGH warning 会暂停安装，MEDIUM warning 会要求交互确认， LOW warning 会提示后继续安装。

HIGH

暂停安装。只有在确认风险来源之后，才应该用 --ignore-warning 继续。

MEDIUM

交互终端会询问 y/n；非交互环境无法确认，因此会暂停。

LOW

默认敏感度下只提示 warning，然后继续安装。

可以通过 --sensitivity 参数提高拦截强度。

检查项

使用 spip install 时，secured_pip 会围绕解析后的安装计划和安装产物运行下面这些检查。

typo-suspect MEDIUM / LOW

在 pip 解析出安装计划后，检查包名是否像热门包的“假拼写错误”。secured_pip 会把不热门的已解析包名和本地热门包列表进行比较。

例如 requsets 对 requests、panda 对 pandas，以及 sixth 对 six。

direct-url MEDIUM

当安装目标或已解析依赖使用 direct URL、VCS URL，或者 PEP 508 direct reference，而不是普通 registry 包引用时报警。

recent-release MEDIUM

当选中的 PyPI release 发布时间不到 2 天时报警，因为非常新的 release 留给社区审查的时间更少。

empty-description LOW

当选中的 PyPI release 元数据里 summary 和 long description 都为空时报警。

suspicious-url LOW

当 PyPI 元数据指向短链接、裸 IP、可疑顶级域名、嵌入凭据，或类似的可疑 URL 模式时报警。

repository-mismatch LOW

当 PyPI 元数据指向 GitHub 或 GitLab 仓库，但仓库名看起来和包名无关时报警。

email-domain-drift LOW

当包维护者邮箱域名和 secured_pip 本地历史缓存中记录的域名相比发生变化时报警。

zero-version LOW

当选中的包版本是 0.0 或 0.0.0 时报警。

suspicious-pth MEDIUM

检测新增或被修改的 .pth 文件中是否存在以 import 开头的可执行行。报警后会询问是否删除可疑的已安装 .pth 文件。

工作流

推荐工作流很简单：继续输入你熟悉的包安装命令，但让它们经过 secured_pip，这样 guarded install 路径就能检查解析后的安装计划和准备好的本地安装产物。

对于 pip install . 这类本地安装路径、基于本地文件的安装，或者显式使用 --no-index、--find-links 的离线安装，secured_pip 会禁用 registry 元数据请求。此时它会专注于 direct install 检测和 .pth 检查，而不是去查询远端包索引的元数据。

这里禁用的只是远端元数据查询，不会关闭本地安全检查。本地路径、安装产物内容，以及安装后的 .pth 变化仍然会继续检查。
