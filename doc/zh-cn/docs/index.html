介绍

secured_pip，也就是 spip，是一个轻量的 pip wrapper。它保持普通 pip 的行为，只在 pip install 周围加入供应链安全检查。

概览

spip 面向日常 Python 安装路径设计。除 install 之外的命令都会转发给 python -m pip。对于 spip install，解析、下载、构建和安装仍然走 pip；spip 只在关键节点插入轻量检查。

安装

先安装 secured_pip，再把 pip alias 到 spip，这样原来的工作流不用改变。

cmd（Windows）

pip install secured_pip
doskey pip=spip $*

Bash（Linux）

pip install secured_pip
echo "alias pip='spip'" >> ~/.bashrc
source ~/.bashrc

Zsh（macOS）

pip install secured_pip
echo "alias pip='spip'" >> ~/.zshrc
source ~/.zshrc

报警机制

spip 发现的每个风险都会被归入三个等级之一。在默认 low 敏感度下， HIGH warning 会暂停安装，MEDIUM warning 会要求交互确认， LOW warning 会提示后继续安装。

HIGH

暂停安装。只有在确认风险来源之后，才应该用 --ignore-warning 继续。

MEDIUM

交互终端会询问 y/n；非交互环境无法确认，因此会暂停。

LOW

默认敏感度下只提示 warning，然后继续安装。

可以通过 --sensitivity 参数提高拦截强度。

工作流

推荐工作流很简单：继续输入你熟悉的包安装命令，但让它们经过 spip，这样 guarded install 路径就能检查解析后的安装计划和准备好的本地安装产物。

Shell alias 
alias pip='spip'
