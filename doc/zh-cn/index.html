为什么选择 secured_pip？

轻量。

secured_pip 更强调本地化和轻量化。

不需要代理服务，不要求 Docker，也不需要额外控制面，让你几乎感受不到它的存在。

可靠。

它专注于 Python pip 安装链路，而不是一个过宽的多生态安全面。

100% 无痛地帮你 100% 避开 LiteLLM、dspy、colorama 这类供应链投毒风险。

无摩擦。

直接使用 spip，或者把 pip alias 到 spip。

完全不用学习，也不用改变你原来的日常工作流。加上其轻量化的特征，这完全是一个无摩擦的安全保护伞。

开始安装。

先安装 secured_pip，再用 spip 命令。最好把 pip 变成 spip 的别名，这样你原来的工作流完全不用变。

cmd（Windows）

pip install secured_pip
doskey pip=spip $*

Bash（Linux）

pip install secured_pip
echo "alias pip='spip'" >> ~/.bashrc
source ~/.bashrc

Zsh（macOS）

pip install secured_pip
echo "alias pip='spip'" >> ~/.zshrc
source ~/.zshrc

你的 PyPI 供应链安全就这样搞定。

现在当你输入 pip install 的时候，会安全很多 :)
